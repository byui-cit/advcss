Layout: Multi part forms

Professional web developers and designers end up spending a lot of time with forms. As one of the few ways to get direct feedback from our users they play a critical role in web sites. They can also be quite tricky to get right. Begin this activity by reading through Styling HTML Forms and Advanced Styling for HTML Forms on MDN.

So the bad news is that forms can be problematic to style...the good news is that the problems are not new and there are some good techniques around for getting nice looking forms. One of the issues with forms is that they can be overwhelming to users if they ask for too much information. One method of dealing with this is to break the form up into smaller parts. We are going to build a multipart form with this activity.

Activity

  1. Prepare

    Create a new folder on your computer for this project. Add two files to the project called index.html and styles.scss. Add the code necessary to start out a valid HTML5 document to the index.html file. In the head include a link to your .css, and also links to a couple of javascript resources: 

    <script src="https://cdn.jsdelivr.net/gh/cferdinandi/bouncer@1/dist/bouncer.polyfills.min.js"></script>
<script src="https://byui-cit.github.io/advcss/layout/forms.js" defer ></script>

    Those files will provide the functionality to get our form to switch from section to section, and will provide some validation.

  2. Review the design

    The form we will be building resembles closely the one in this multi-step form example from Codepen. Ours will not be animated, but will include some validation. Review it, notice what changes when you hit "next".

  3. Build the Form

    Add a form to your index file with a class of mpform. Our form will have 3 sections, as well as a list to track which section of the form we are currently on. Add an ol with a class of mpform__progressbar. Add three li children for the tracking. Then add three fieldsets, each with a class of mpform__fieldset. Each fieldset will become a step in our multipart form. Your HTML should look similar to this: 

    <form class="mpform">
  <ol class="mpform__progressbar">
    <li class="active">Account Setup</li>
    <li>Personal Details</li>
    <li>Contact</li>
  </ol>
  <fieldset class="mpform__fs"></fieldset>
  <fieldset class="mpform__fs"></fieldset>
  <fieldset class="mpform__fs"></fieldset>
</form>

    Below is a screenshot of the example we are basing our form off of. Note that the first number in the progress bar is green. That is what the active class will do. Notice also that the section of the form has two titles, and three inputs. The example uses placeholder text to indicate what each field is for...we will add labels to ours. Relying on only a placeholder can sometimes cause confusion as the placeholder disappears as soon at you focus the input.

    Add the HTML to the first fieldset. Again...for example mine looks like this: 

    <fieldset class="mpform__fs">
  <h2 class="fs__title centerText">Create your account</h2>
  <h3 class="fs__subtitle centerText">This is step 1</h3>
  <p>
    <label class="fs__label" for="email">Email</label>
    <input type="email" name="email" placeholder="Enter a valid email" />
  </p>
  <p>
    <label class="fs__label" for="pass">Password</label>
    <input type="password" name="pass" placeholder="Enter at least 8 characters" />
  </p>
  <p>
    <label class="fs__label" for="cpass">Confirm Password</label>
    <input type="password" name="cpass" placeholder="Re-enter Password" />
  </p>
  <div class="centerText"><button class="next ">Next</button></div>
</fieldset>

    While we are here we may as well fill out the other two fieldsets. The second should also have three fields: first name, last name, and phone. The last fieldset will have two checkboxes...the first should indicate we want to recieve a newsletter, then second would indicate that we would like to receive special offers...both should be checked by default...because why would someone not want those things!

  4. Progress Bar

    Before we start on the progress bar, let's do some base styling of the form. Let's set a max-width of 400px, then center it. In addition add the following rules to the top of your SCSS: 

    
*, *::before, *::after {
  box-sizing: border-box;
}
html,
body {
  height: 100vh;
  font-family: Arial, Helvetica, sans-serif;
}

    Our progress bar has for it's base an ordered list, but the default numbers for that list do not have the correct styling for the progress bar. So we will need to hide the default numbers, and replace them with a custom counter that we can style using a ::before element. The bars tying the numbers together will be made with an ::after element.

    Below is my base styling for the progress bar:

    
.mpform__progressbar {
  display: flex;
  justify-content: space-around;
  width: 100%;
  padding-left: 0;
  list-style-type: none;
  counter-reset: step;
}

    Hopefully the only thing you don't recognize there is counter-reset: step;. The line above removes the default numbering, then that line initializes our custom counter. This is the first step. The other lines are to arrange our steps in a row and center them in the space. 

    
.mpform__progressbar > li {
  flex: 1 0 33%;
  text-align: center;
  position: relative;
  &::before {
    content: counter(step);
    counter-increment: step;
    margin: auto;
    border-radius: 4px;
    font-size: 1.2em;
    display: block;
    width: 30px;
    line-height: 30px;
    text-align: center;
    background-color: $step-inactive-color;
  }
}

    The next step is to style the items and the ::before. In the ::before CSS you will see the rest of what we need to display custom counters. First we see content: counter(step). This is responsible for actually displaying our new numbers. Without the next line we would see the same number on each item. The word step could be changed to anything we want really, but whatever we use in counter-reset determines what we use for the rest as well. Read about custom counters on MDN to learn more.

    I have defined a few SCSS variables with some colors. Add the following to the top of your SCSS file (you can change the actual colors to whatever you want): 

    $step-active-color: green;
$step-inactive-color: antiquewhite;
$step-active-font-color: white;

    Your progress bar should look something like this:

    We now need to create the connecting bars, and add the active styles. We will use the ::after pseudo-element to do this. Add the following SCSS to your file: 

    
&::after {
  content: "";
  position: absolute;
  top: 13px;
  left: -50%;
  height: 4px;
  background-color: $step-inactive-color;
  width: 100%;
  z-index: -1;
}

    Those lines will do several things...first it will make a skinny colored rectangle that is as wide as the LI elements. Second we use position:absolute to place the bar into the correct place. Because we want the bar behind the current active step number to be lit up we move all the bars left. So the bar for number 2 is left of the 2. Notice also that position:relative was set on each list item. So our absolute coordinate system is relative to the list item. Finally we use z-index:-1 to make sure the bar is behind everything else.

    The bars for 2 and 3 are looking good...but we have a bar for 1 that we don't need or want. Add the following CSS to remove that bar: 

    
&:first-child::after {
  content: none;
}

    The & in this case refers to .mpform__progressbar > li. (Those lines should be nested under that selector) So this would read "Find the li that is the first child of it's parent and remove the bar from the after by setting it's content to nothing."

    Finally we need a rule to change the color of the active step and all steps previous. The following should do it: 

    
  &.active:before,
  &.active:after {
    background: $step-active-color;
    color: $step-active-font-color;
  }

    We only need to add a class of active to a li step for it to change color now. The first step should already have that class on it. The Javascript that was provided above will take care of adding and removing that class as necessary for the other steps.

  5. Styling the Fieldsets

    In order to have our form match the model we need to clean up our inputs. They should be the same size, centered, bigger (around 40px at a minimum for a good mobile target size), we are also going to add labels and would like them to overlap slightly with the input they describe. The image below shows our goal. If you would like to figure it out on your own, stop reading and go do it! Then compare what you did with my suggestions. It's ok if your solution does not exactly match mine. Otherwise continue for my explanation.

    We can start with the following general fieldset rules:

    
.mpform__fs {
  width: 95%;
  margin: 2em auto;
  border: 1px solid #ccc;
  border-radius: 5px;
  box-shadow: 0 0 15px 1px rgba(0, 0, 0, 0.4);

  > h2 {
    margin-bottom: -0.5em;
  }
}

    Hopefully there are no surprises in the css above. You have probably noticed that I am not explaining every line. Much of it I assume you have seen before and understand. However, if there is any of the CSS in this activity you don't understand and that I don't explain, STOP and figure out both what the lines do, and why we used them in this instance before moving on!

    To style the inputs we could build a selector like this: .mpform__fs input, but we will have some inputs of type="checkbox" later, and we don't want these styles to apply to those elements. So let's use some attribute selectors to identitify exactly which types of inputs we want ot style right now. (This chould be nested under .mpform__fs 

    
  input[type="text"],
  input[type="password"],
  input[type="email"] {
    width: 100%;
    font-size: 1em;
    padding: 0.8em;
    margin-bottom: 0.5em;
    border: 1px solid #ccc;
    border-radius: 3px;
  }

    That will do for a start. It is safe to assume that our text, email, and password inputs will all be styled similarly. Look over each line to make sure you understand what each is doing. Notice that we have used almost the exact 2 lines of CSS twice now: 

    border: 1px solid #ccc;
border-radius: 5px;

    We may need more rounded boxes too. Let's create a mixin to do that. 

    
@mixin borderWithRadius($radius: 0, $color: #ccc) {
  border: 1px solid $color;
  border-radius: $radius;
}

    then replace those lines where they occur in our SCSS. Our rule for .mpform__fs should look like this now: 

    
.mpform__fs {
  width: 95%;
  margin: 2em auto;
  @include borderWithRadius(5px);
  box-shadow: 0 0 15px 1px rgba(0, 0, 0, 0.4);

  > h2 {
    margin-bottom: -0.5em;
  }

  input[type="text"],
  input[type="password"],
  input[type="email"] {
    width: 100%;
    font-size: 1em;
    padding: 0.8em;
    margin-bottom: 0.5em;
    @include borderWithRadius(3px);
  }
}

    The next step will be to style the button, and get things centered. (The HTML provided above had a class of .centerText on certain elements already, we just need to write the class.)

    
  > .centerText {
    text-align: center;
  }
  button {
    background-color: $step-active-color;
    color: $step-active-font-color;
    border: 0;
    font-size: 1em;
    padding: 0.8em;
    width: 25%;
  }
