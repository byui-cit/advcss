Shapes with SVG

What is SVG?

SVG stands for Scalable Vector Graphics. It is actually an XML format where the image is described through a series of vectors (If you remember from Algebra, a vector is just a line with direction) Most other image formats are bitmap: a collection of colored dots. Since SVG is just a bunch of lines, and lines can easily be lengthened and shortened, SVG images have the advantage of being resolution independent. They scale up and down with no degradation in quality.

The other really cool thing about SVG graphics is that they can be modified with CSS!

If you have not yet done so...take some time right now and do the reading this week on SVG: Jump Start HTML: Ch17-18 and then look through the W3Schools SVG tutorial

Activity

SVG is a large topic that we don't have time in this class to fully explore. My goal is to help you learn enough to be comfortable using an SVG graphic in a webpage. With that in mind lets start with some review from the reading.

Adding an SVG image to your page

The reading mentioned several ways to get an svg added. You can add them as an <object>, an <embed>, an <img>, as a background image in css, or inline using the <svg> element

To keep things simple I am going to focus on the inline method using <svg>

The W3Schools SVG tutorial gives this as it's first example:


            <svg width="100" height="100">
              <circle cx="50" cy="50" r="40" stroke="green" stroke-width="4" fill="yellow" />
            </svg>

This introduces us to the <svg> element. I would recommend that you copy the code snippet above into Codepen and follow along and experiment with me. It will be really important that you try this...don't just read the activity...do it!

Notice the width and height set on it. Width and height with svg takes a bit of explanation. You will get part of that now and part later. Change the width to 120. Did the result match what you expected? Probably not. The width and height set on the <svg> element sets the size of the container that our image will be drawn in. If we set a border on it you can see this more clearly. Lets do that.

We mentioned that one of the advantages of SVG is that you can modify them with CSS. Lets try it. Modify your html to look like this:


            <svg class="svgcontainer">
              <circle cx="50" cy="50" r="40" stroke="green" stroke-width="4" fill="yellow" />
            </svg>

And add this to your CSS in Codepen:


            .svgcontainer {
                width: 100px;
                height: 100px;
                border: 1px solid;
            }

Try changing the values on the circle now. Change r to 60 for example. Now what happened? Your circle is bigger than the box now.

Combining shapes (circle, line, rect, path)

You should have noticed from your reading that circle is not the only shape SVG provides us with. There are many others I encourage you to play around with including rectangle, polygon, line, and path. Their shapes can be combined as well. For example add the following to your html in Codepen.


            <svg width="400" height="180">
              <rect x="50" y="20" rx="20" ry="20" width="150" height="150" class="rectangle"/>
              <circle cx="125" cy="95" r="30" stroke="darkred" stroke-width="4" fill="white" />
            </svg>

We will need the following css added as well.


            .rectangle {
                  fill:darkred;
                  stroke:black;
                  stroke-width:5;
                  opacity:0.5;
            }

Patterns

Patterns are another way to group svg objects. they have a specific purpose as well. To create background patterns to use in other SVG objects. Lets take a look at an example. We will use the square shape we made earlier and change it to a pattern. In order to use patterns we need to add a new section to our svg...the <def>.


 <svg width="200" height="200"> 
    <!-- define pattern -->
    <defs>
      <pattern id="mypattern" patternUnits="userSpaceOnUse" x="0" y="0" width="50" height="50">
        <rect x="0" y="0" width="50" rx="10" ry="10" height="50" class="rectangle" /> 
        <circle cx="25" cy="25" r="8" fill="white" stroke="darkred" stroke-width="2" />

      </pattern>
    </defs>

    <!-- use pattern in a circle -->
    <circle cx="100" cy="100" r="100" fill="url(#mypattern)" stroke-width="1" stroke="black" />
</svg>

Complex SVG

SVG can be used for very complex images. Most of those are not made by building them by hand however. They are made with an editor. The most popular SVG editor is Adobe Illustrator. Another good but free option is called Inkscape. Drawing with vectors to create complex SVG images is not for the faint of heart however! There is a pretty big learning curve (If you are interested look into Comm 305: Vector Graphics). Luckily there is a lot of SVG available out in the internet that is free to use. Lets look at what it might take to get it onto our page.

For this I am going to use resources available at icomoon.io. This site offers a lot of good free icons you can use. Go ahead and select a few you like then click on the "Generate SVG & More" button. After that hit the Download button and browse to your downloads folder to find the svg you requested.

In the folder that downloaded you actually have 3 options for how to include the downloaded svg icons. The first is inline, just like I have done above, almost. If you look at the demo.html file you can see examples of how to do that.

Notice that all the icons are defined together using the <symbol> tag. Here is an example of one of them.


<symbol id="icon-plane" viewBox="0 0 1024 1024">
<title>plane</title>
<path class="path1" d="M496.672 125.12c26.496 0 48 21.504 48 48v248l320 200v86.016l-320-108v168l65.248 64.928-0.032 66.688-113.216-37.632-111.584 37.76 0.48-70.048 61.12-59.68 0.544-169.984-319.904 109.952 0.672-88 320.224-202.016 0.512-246.016c-0.064-26.464 21.44-47.968 47.936-47.968z"></path>
</symbol>

To use the svg later in the document you would take advantage of the <use> element. Here is another example.


        <svg class="icon icon-plane">
          <use xlink:href="#icon-plane"></use>
        </svg>

This is great for reusability. The svg only has to be defined once, but can be used as many times as needed with this method.

The second way would be to take advantage of the .svg files in the SVG folder. Those can be used just like any other image. Added as the src in an <img> tag or as a background image in css. The biggest disadvantage of adding SVG as an image is that you cannot style them with css. You can style background images however.


            <img src="svg/plane.svg" alt="SVG image of a plane">

...or...


            background: url("svg/plane.svg");

The third way can be seen in the demo-external.html file. All the SVG is in an external SVG file called symbol-defs.svg. You can see how they are loaded in the .html file. This method has several advantages such as separation of the images from the html and enabling caching of the svgs.

Compatibility

The basic functionality of SVG has very broad support in all the major browsers. There are a few things to watch out for however.

External SVG with the <use> element is not supported in Internet Explorer. You would have to use a polyfill if you wanted to take advantage of those methods of including svg. One was actually included with the bundle we downloaded from Icomoon...svgxuse.js
