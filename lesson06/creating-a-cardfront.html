Creating a card front layout

CSS Layout

We have seen many layout methods come and go for CSS. All of them have had their shortcomings. Picking a layout method has often felt like picking the lesser of many evils with CSS! The good news is that things are improving. Two recent additions to CSS are Flexbox and CSS Grid. CSS grid is just ready for widespread use, provides a nice complement to flexbox. Both together solve many long standing problems with other layout methods.

Flexbox is ideally suited for layouts that flow either horizontally or vertically (which happens a lot with the current trends) and can simplify responsive design with the ease at which it can switch between those two flows.

CSS Grid excels with two dimensional layouts...where both the X and the Y position of an element are critical.

Last week we created a custom playing card back, continuing along that theme this week we will be creating the front using Grid and Flexbox. We are going to start with the 7 of Clubs. (Actually...you could make a 7 of anything if you wanted. I once even made a Seven of foxes :) )

Tips and Guidelines

  • You should be able to reuse some of the classes from the card back assignment last week. For example the classes that form the card outline could stay the same.
  • Many versions of the 7 of clubs (and other cards) will have a kind of half row where the center pip is wedged in between 2 other rows. Don't worry about re-creating that. Go ahead and give each row of pips on the card a full row in your layout.
  • Along those lines...don't stress about making your html card look exactly like whatever 7 of clubs you are looking at. As long as it looks good, is balanced, and has all the elements you will get credit.
  • You can use either a css shape or svg for your suits. If you want to use svg I have an example file with the svg suit shapes in it to start with.
  • A playing card is a good example of a grid based layout. In such layouts everything should fit in one or more defined columns. Sometimes in such layouts columns need to be skipped. In grid terminology a skip is called an offset. This assignment becomes easier if you create yourself some offsets...I would say offset-1 and offset-2 (skip one or two columns) would be sufficient in this case. How do you accomplish this? Well, if you needed to push anything left a small distance, how would you do it?

Using CSS Grid

Grids and web design have a bit of a complicated history. This is largely due to the fact that we keep using the same word to mean different things. For example designers have long used grid theory to help in their designs (Please stop here and go read that article! Then come back). If you do a seach for "grid libraries" you will find a ton of tools that have been built in CSS (usually using float or flexbox) to help take your design based on grid theory and build it into a grid based layout.

Recently the W3C finished the CSS Grid Layout Module Level 1 specification and we are rapidly seeing browser support. This specification is the second part of the layout solution web developers have been awaiting for many years. We got the first with Flexbox...but flexbox really only shines with one dimensional layouts...either a row or column of content that may or may not wrap. Grid is the answer for layouts where X and Y position is critical.

Keep in mind that grid design theory, grid libraries, and the CSS Grid specification are different things even though they share similar names. From here on when I refer to Grid...I mean the specification.

For the first part of this activity we will build a card front layout with CSS Grid.

  1. Create (or Fork) a pen

    Create a new pen in codepen.io (or if you would rather you can fork this starter card pen and use it as a starting point). Make sure to change the css settings to use SCSS so you can take advantage of Sass. You should be able to use some of the classes you created for the Card Back assignment to create the general shape of the card. Continue by adding all the elements you will need next.

    For example we are making the 7 of clubs which looks something like this:

    I would need a number 7 and small suit, then 7 regular sized suits, then another 7 and small suit. Notice that some of those icons are upside down! Try this class to make that happen: 

    
.rotate {
transform: rotate(180deg);
}

  2. Create a Grid based layout for the Card

    Create a CSS Grid based layout for your card. Your grid will work best if all of the pips on the card share a common parent. Grid is similar to Flexbox in that when you turn on grid for an element...it only effects the immediate children of that element.

    Adding Wider Support

    Hopefully your Grid layout came together easily and you are beginning to understand the power behind this specification. While it has pretty good support, there are still a lot of older browsers that do not support it yet. You should always have a fallback plan with your sites, especially when using newer features.

    For the rest of this activity we will continue with the Grid based card created earlier. We are going to enhance it so that if viewed in a browser that supports Grid, it will use Grid for the layout, otherwise it will use Flexbox.

  3. Working with @supports

    Open your Grid card pen in codepen.io. Add a @supports not {} block checking to see if the browser does not support Grid. Add the SCSS you would need to rebuild your card with Flexbox.

    While building, if you are in a browser that does support Grid you may have a hard time seeing your Flex layout to make sure it is right. You may find it easier to check if Grid is supported for testing, then switch it to a not supported at the end.

    Initially (while building and testing) your supports block will probably look something like this:

    
							@supports (display: grid) {
								All of your flexbox SCSS code here.            
							}

    Before you turn it in however change it to:

    
								@supports not (display: grid) {
									All of your flexbox SCSS code here.            
								}

    @supports works very similarly to @media. Both take a condition and the css in the block associated with them is run if the condition is true. Like @media, you need to make sure that you have the default css styles before the @supports block in your file.

  4. Flex based layout

    Create a Flexbox based layout for your card. You should try to not add any additional HTML markup to your card. Doing so will most likely break your Grid layout.

  5. Verify and Submit

    Remove the "not" from the @supports block to test that you did not break your Grid layout while making your Flexbox one. Submit the URL to your codepen to Ilearn.
