Using CSS Grid

Grids and web design have a bit of a complicated history. This is largely due to the fact that we keep using the same word to mean different things. For example designers have long used grid theory to help in their designs. If you do a seach for "grid libraries" you will find a ton of tools that have been built in CSS (usually using float or flexbox) to help take your design based on grid theory and build it into a grid based layout.

Recently the W3C finished the CSS Grid Layout Module Level 1 specification and we are rapidly seeing browser support. This specification is the second part of the layout solution web developers have been awaiting for many years. We got the first with Flexbox...but flexbox really only shines with one dimensional layouts...either a row or column of content that may or may not wrap. Grid is the answer for layouts where X and Y position is critical.

Keep in mind that grid design theory, grid libraries, and the CSS Grid specification are different things even though they share similar names. From here on when I refer to Grid...I mean the specification.

Earlier this week we created a playing card front with flexbox, now we will do the same with Grid.

You need to make sure that you are on a version of your browser that supports Grid. Check the Caniuse link above to verify.

Activity

For this activity we will continue with the flexbox based card created earlier in the week. We are going to enhance it so that if viewed in a browser that supports Grid, it will use Grid for the layout, otherwise it will use Flexbox.

Resources

We will need some additional resources with knowledge about Grid and @supports to accomplish this task. Review the following:

  • Grid By Example Rachel Andrew's great tutorial site for Grid.
  • Introduction to the CSS Grid Layout Module Sitepoint article
  • Complete Guide to Grid CSS Tricks
  • Introduction to CSS's @supports Sitepoint Article

Step 01:  Open your flexbox card pen in codepen.io. Add a @supports block checking to see if the browser supports Grid. Add the SCSS you would need to redo your card with Grid. (This means you will also need to undo some things.)

Step 02:  You may find it easier to take all of your flexbox based code and lock it up in a @supports(not()) block...have the Grid be the default, and the Flexbox used only if Grid isn't supported than the other way. That way you get to start with more of a clean slate with your Grid.

Step 03:  Submit the URL to your codepen to Ilearn.
