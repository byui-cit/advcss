Create your own grid

As you should have noticed in the reading for this week designing to a grid has become very popular. As a result there are a lot of css libraries created to help you build the web pages that you have designed with your grid. We will look at some of those later in the semester, but for this week I would like you to have the experience of building your own specialized grid framework.

Activity

This activity will have you create a css grid library. To keep the scope reasonable we are going to build a specialized library that will be used for a specific purpose: to layout and build playing card fronts.

In other assignments I have provided html for you to style that you were not allowed to change. In this case I should be able to take your .scss and using only the classes you provide in your library I should be able to write the html to build any card from A-10 without changing any css!

Tips and Guidelines

  • A typical playing card has a height that is 1.4 times it's width.
  • Your library should use flexbox for positioning.
  • One of the first (and arguably most important) decisions you will need to make is how many columns should you have in your grid.
  • You will probably need to create a class to create a row in your grid, and then classes to create 1 column, 2columns, etc.
  • Classes to create offsets (skipping columns) would probably also be helpful.
  • For the suits you can use images, css border shapes, or svg. If you want to use svg I have an example file with the svg suit shapes in it to start with.
  • If you get stuck try poking around in an existing grid framework to see how they solved certain problems. Make sure the one you look at uses flexbox and not floats or inline-block however.
  • Work together! Don't do each others work, but take advantage of your classmates to bounce ideas around and talk through the problem.
  • If you are having a hard time getting flexbox to work for you make sure to spend some time with both CSS Tricks excellent Guide to Flexbox and Flexbox Froggy. It really will help you get all the properties and how they work together down.

Step 01:  Create a new pen in codepen.io. Make sure to change the css settings to use SCSS so you can take advantage of Sass. You should be able to use the same classes you created last week for the card backs to create the general shape of the card. Choose a card to start with (I recommend 5, 7, or 10) and add all the elements you will need.

For example if I were making the 5 of clubs which looks something like this:

I would need a number and small suit, then 5 regular sized suits, then another 5 and small suit. Notice that some of those icons are upside down! Try this class to make that happen: 


.rotate {
    transform: rotate(180deg);
}

Step 02:  Create the rest of the classes you will need to make your card look right. Then create a second different card to test your library. Hopefully you won't need to make any changes to your css to get it to work, but make adjustments as necessary. Once the first and second card are correct try as many others as you need to feel comfortable that your library will work for any card A-10. (Don't forget to try the Ace ;) )

Step 03:  Submit the URL to your codepen to Ilearn.

Grading

  • Grid library can be used to create any playing card A-10 by changing the html markup but without changing the css. 10pts
  • SCSS has been used where appropriate. (variables for colors or sizes would be a good start) 4pts
  • Grid is Flexbox based. 6pts
