Using grid layout systems

If you do any looking on the web, you will quickly find that designing to a grid has become very popular. As a result there are a lot of css libraries created to help you build the web pages that you have designed with your grid. Earlier this week we took a look at flexbox and solving a particular problem with it. Now we get to solve it again...but this time with a grid system/library.

If you do a search for "css grid library" you will be overwhelmed with choices. (Do not confuse this with the CSS grid specification that I mentioned briefly in the last assignment they share similar names...but are not the same.) It seems everyone has taken a stab at solving this particular problem in their own way. Most of them work very similarly however, and learning to use one makes it pretty straightforward to pick up another later.

Activity

For this activity we will be using a flexbox based grid system called Flexbox Grid. We will be creating the same card (a 7 of clubs) that we created in the Flexbox assignment. As you complete this pay attention to the similarities and differences between the 2 approaches.

One that I will bring to your attention right now is that when using a css framework most of the challenge is in attaching the correct class(es) to the correct elements in the HTML. You actually shouldn't write very much of your own css for this activity. If you find yourself writing a lot of the CSS you are working too hard!

The link I provided above has the download button front and center. There is also an online CDN (content delivery network) version you can use as well. This makes it convenient if you want to use a service like codepen to do your work.

To allow an external css file to work in codepen you will need to goto the css settings (Little gear icon next to the CSS section) then under the "Add External CSS" section paste the following:

https://cdn.jsdelivr.net/flexboxgrid/6.3.0/flexboxgrid.min.css

After adding that you will be able to use all the css from the library.

Tips and Guidelines

  • You should be able to reuse some of the classes from the flexbox assignment for this one. For example the classes that form the card outline, and size and color the symbols should stay the same.
  • Most Grid systems (including Flexbox Grid) use a 12 column grid. This is not ideal for a playing card...but with a little creativity you can make it work.
  • Just like with the flexbox assignment you will need to decide how many rows you will use. I have successfully completed this with everything in one row and each row on the card in a separate <div class="row"> as well. Each has different challenges.
  • You will probably want to read the documentation...no really...read the docs and look over the examples ;) They are the text and tutorial for this activity and learning to navigate documentation for something like this might just be the most valuable thing you take from this activity.
  • Don't worry about the half rows many cards use. Each row of pips should have the same amount of space.
  • Remember that you can turn a border on for your <div>s. This can help to see what is going on when working on alignment.
  • Finally don't be afraid to look at the actual css for the library. You can see how they did everything! Sometimes this can help you to understand why it is doing (or not doing) something.

Getting Started

If you have never used a framework before it can be a bit daunting at first. Where do you start?

All grid libraries work by creating rows and columns. Begin by determining what classes the library expects you to add to your html to identify these. For example Flexbox grid starts out with an example that looks something like this (I've added a bit to their example):


<div class="row">
    <div class="col-xs-12
                col-sm-8
                col-md-6
                col-lg-4">
        <div class="box">Box1</div>
    </div>
    <div class="col-xs-12
                col-sm-4
                col-md-3
                col-lg-4">
        <div class="box">Box2</div>
    </div>
    <div class="col-xs-12
                col-sm-6
                col-md-3
                col-lg-4">
        <div class="box">Box3</div>
    </div>
</div>

We can learn a lot about this library by looking closely here. It starts with a div with class="row". I think we can safely assume that we let the library know where we need a "row" by adding this class. For this framework a row is made up of 12 columns. We should try to keep our rows to this limit...but if we exceed it the row will just wrap at 12. Sometimes we can take advantage of this behavior.

Next is a child div with 4 classes on it. This is how we tell the framework how many columns this element should take up in the row. This type of pattern is pretty common in responsive frameworks. We have told the element: "Be 12 columns (use the whole row) on a really small screen, be 8 columns wide on a small screen, be 6 columns wide on a medium screen, or be 4 columns wide on a large screen." (To figure out what actual sizes these are you'll need to dive into the code.)

If you only want to designate one set of sizes use the xs one (Mobile first!). If you used the medium for example your layout would work for medium and larger screens, but if you got smaller than that your layout would break.

So for our example our 3 boxes would behave like this at each screen size:

  1. xs: each box on it's own row taking up 100% of the row
  2. sm: first 2 boxes sharing the row. box1 taking up 2/3 (8 cols) and Box2 the other 1/3 (4 cols). Box3 would be on the next row...width 1/2 (6 cols)
  3. md: all 3 boxes on the same row...Box1 gets half the space, Box2 and 3 split the other half evenly.
  4. lg: all 3 boxes on the same row each getting 1/3 (4cols) of the space.

For this exercise you really only need to specify the xs columns.

The term with these libraries when you want to skip one or more columns is "offset".
