Adding Wider Support

Hopefully your Grid layout came together easily and you are beginning to understand the power behind this specification. While it has pretty good support, there are still a lot of older browsers that do not support it yet. You should always have a fallback plan with your sites, especially when using newer features.

This week we will take our CSS Grid based card from Part 1 of this activity and using some conditional CSS magic provided by @supports we will re-create our layout using flexbox for those browsers that do not support grid.

  • Working with @supports

    Open your Grid card pen in codepen.io. Add a @supports not {} block checking to see if the browser does not support Grid. Add the SCSS you would need to rebuild your card with Flexbox.

    While building, if you are in a browser that does support Grid you may have a hard time seeing your Flex layout to make sure it is right. You may find it easier to check if Grid is supported for testing, then switch it to a not supported at the end.

    Initially (while building and testing) your supports block will probably look something like this: 

    
@supports (display: grid) {
	// All of your flexbox SCSS code here.            
}

    Before you turn it in however change it to:

    
@supports not (display: grid) {
	// All of your flexbox SCSS code here.            
}

    @supports works very similarly to @media. Both take a condition and the css in the block associated with them is run if the condition is true. Like @media, you need to make sure that you have the default css styles before the @supports block in your file.

  • Flex based layout

    Create a Flexbox based layout for your card. You should try to not add any additional HTML markup to your card. Doing so will most likely break your Grid layout.

  • Verify and Submit

    Remove the "not" from the @supports block to test that you did not break your Grid layout while making your Flexbox one. Submit the URL to your codepen to Ilearn.
