Building a Simple Component

Websites are often just collections of pieces called components. Treating them this way helps us to write more re-usable code. It also can help reduce the complexity of redesigns. We will read and discuss this more next week. This week however I would like you to produce the HTML and CSS for a simple web component.

Activity

  1. Review the layout.

    Below is a screenshot of the component we would like to build. It is a simple shopping cart. Review it considering how the HTML should be structured, and how you might layout the different elements.

  2. Write the HTML

    Create a new Codepen and write the HTML. Keep in mind semantics as you choose the elements to represent each part of the cart. Also keep in mind how you should structure things to make your layouts work. i.e. if you are using Flexbox or Grid do all of the right elements share a common parent?, etc.

  3. Style

    Add your SCSS now to replicate the screenshot as closely as possible. Pay attention to padding, margins, font-sizes, placement, etc. You can just use helvetica, arial, san-serif for the font.

  4. Submit in Ilearn

    Submit the codepen url to Ilearn.

Grading

  • Screenshot replicated closely in HTML and SCSS. 8pts
  • Attention paid to semantics in HTML. 2 pts
