Create your own grid

We've now looked at 2 different ways to build the layout for a playing card. Once from scratch, and once using a grid system library. While building the card layout you probably noticed that the library we were using was not ideally suited for the task we had.

Funny enough this is often the case, even when we are not building something as specific as a seven of clubs. The reason there are so many grid systems out there is that they are never perfect for every task. It is not uncommon to have to crack open the hood, so to speak, and tinker. This week I would like to build on what you learned in the 2 activities from last week to create our own specialized grid library.

Activity

This activity will have you create a css grid library. To keep the scope reasonable we are going to build a specialized library that will be used for a specific purpose: to layout and build playing card fronts. I want to state right here that this is not the only way to solve this problem. You will probably see places where this could be improved...and you are welcome to do so. My goal is to give you an introduction to some important concepts in Sass, css, grids, and responsiveness.

In other assignments I have provided html for you to style that you were not allowed to change. In this case I should be able to take your .scss and using only the classes you provide in your library I should be able to write the html to build any card from A-10 without changing any css!

Based on what we have learned here are a few guidlines and tips we will follow:

  • A typical playing card has a height that is 1.4 times it's width.
  • Our library should use flexbox for positioning.
  • One of the first (and arguably most important) decisions we will need to make is how many columns we should have in the grid.
  • We will probably need to create a class to create a row in your grid, and then classes to create 1 column, 2 columns, etc.
  • Classes to create offsets (skipping columns) would probably also be helpful.
  • For the suits we will use svg. I still have an example file with the svg suit shapes in it to start with.
  • You can learn a lot about grids by poking around in an existing grid framework to see how they solved certain problems. Make sure the one you look at uses flexbox and not floats or inline-block however.

Notice that I have a div that will form the border of the card, then I have another just inside that. One of the decisions we will have to make is whether to put all our pips inside one flex parent, or create flex parents for each row of pips. Both work, and each has it's advantages and disadvantages. For this demo I chose to put all the pips into one flex parent. That inner div will become that parent.

Step 02:  Create a class called row and add the css to make it a flex container. Should look something like this: 


.row {
  display: flex;
  flex-flow: row wrap;
}

the rest of the classes you will need to make your card look right. Then create a second different card to test your library. Hopefully you won't need to make any changes to your css to get it to work, but make adjustments as necessary. Once the first and second card are correct try as many others as you need to feel comfortable that your library will work for any card A-10. (Don't forget to try the Ace ;) )
