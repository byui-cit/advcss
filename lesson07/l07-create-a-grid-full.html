Create your own grid

We've now looked at 2 different ways to build the layout for a playing card. Once from scratch, and once using a grid system library. While building the card layout you probably noticed that the library we were using was not ideally suited for the task we had.

Funny enough this is often the case, even when we are not building something as specific as a seven of clubs. The reason there are so many grid systems out there is that they are never perfect for every task. It is not uncommon to have to crack open the hood, so to speak, and tinker. This week I would like to build on what you learned in the 2 activities from last week to create our own specialized grid library.

Activity

This activity will have you create a css grid library. To keep the scope reasonable we are going to build a specialized library that will be used for a specific purpose: to layout and build playing card fronts. I want to state right here that this is not the only way to solve this problem. You will probably see places where this could be improved...and you are welcome to do so. My goal is to give you an introduction to some important concepts in Sass, css, grids, and responsiveness.

When you are done with this activity I should be able to take your .scss and using only the classes you provide in your library I should be able to write the html to build any card from A-10 without changing any css!

Here are a few guidelines and tips:

  • A typical playing card has a height that is 1.4 times it's width.
  • Our library should use flexbox for positioning.
  • One of the first (and arguably most important) decisions we will need to make is how many columns we should have in the grid.
  • We will probably need to create a class to create a row in your grid, and then classes to create 1 column, 2 columns, etc.
  • Classes to create offsets (skipping columns) would probably also be helpful.
  • For the suits we will use svg. I still have an example file with the svg suit shapes in it to start with.
  • You can learn a lot about grids by poking around in an existing grid framework to see how they solved certain problems. Make sure the one you look at uses flexbox and not floats or inline-block however.

Step 01:  Create new .html and .scss files in your editor. You should be able to use the same classes you created last week for the card backs to create the general shape of the card, or you can use my code from below. By the end of this activity we should easily be able to create any card, but let's start with the 7 like we did last week. Add all the elements in HTML you will need for a 7 of clubs. (You may just be able to use the html from last week)

The way you write your html can make a big difference in how your css turns out. Because I will be making a few assumptions during this walkthrough lets take a look at how I chose to write my html.

See the Pen starter card by Shane T (@matkat) on CodePen.

Notice that I have a div that will form the border of the card, then I have another just inside that. One of the decisions we will have to make is whether to put all our pips inside one flex parent, or create flex parents for each row of pips. Both work, and each has it's advantages and disadvantages. For this demo I chose to put all the pips into one flex parent. That inner div (the one with the class of 'row') will become that parent.

Notice also that I have added classes to get all of my sizes correct. .icon takes care of the pips, and .number handles the special cases of the top and bottom corners.

Notice as well that all my sizes are done with percentages and multiples. This will make our card fairly responsive when we are done.

Feel free to copy/paste the html and SCSS from the pen into files in your editor to use as a starting point for this activity.

Step 02:  Create a class called row in your SCSS file, and add the css to make it a flex container. Should look something like this: 


.row {
  display: flex;
  flex-flow: row wrap;
}

You should see an immediate change after adding that. Your pips are probably all lined up in a row that is wrapping. You may have noticed that there is also a class of cell on each pip. We will use this to attach styles that we want to apply to each cell. Right now let's use it to set each cell to be the same width, and center the contents. Your rule should look something like this:


.cell {
  flex: 1;
  text-align: center;
}

Now we have a decision to make. Each grid system is based around a certain number of columns. The ones we used last week used 12 columns to form a full row. Twelve was not ideal for a playing card. We made it work, but since we are making our own let's make it ideal for the task. The question remains however, how many columns should we use for our card?

If we look at the card below you might be tempted to make each pip take up a column. There are never more than 2 pips on a row for any card, so 5 should work right?

The problem is that if you look closely the columns on the outside edges are smaller than the others. Look at the image below for proof

Our card will work much more smoothly if we make an 8 column grid.

Step 03:  Start making the classes for our framework by dividing the full width of the card (100%) by 1, 2, 3, etc to find out how big each portion should be. Then make rules for each.

For example for something that only needs to take up one column (like the number of the card) we would do 100%/8 = 12.5%. Each column should be 12.5% of the full width. The rule using flexbox to make this might look something like this:


.col-1 {
  flex: 0 0 12.5%;
}

Remember with the flex property the first 0 means don't let this flex object grow, the second 0 means don't let it shrink. You won't get the results you want without those values.

Step 04:  Once you have those classes built apply them to the correct cells. For example the first div with the number should be one column wide, and the second and third should be 2 columns wide. The third should be 8 columns wide. It is the only thing on that row!

Your code might look like this:


<div class="row">
        <div class="number cell col-1">5
            <svg class="icon">
                <use xlink:href="#icon-clubs"></use>
            </svg>
        </div>
        <div class="cell col-2">
            <svg class="icon">
                <use xlink:href="#icon-clubs"></use>
            </svg>
        </div>
        ...

Note!

The class you apply to each pip describes how many columns that element should span...not which column the element is in. Since we only have elements that take up 1, 2, or all of the columns you will only be using those 3 classes (.col-1, .col-2, .col-8 ) for this card.

After getting all of those classes applied your card should look something like this:

It's just not looking right. It's missing the spacing. There should be gaps between the pips that we just don't have yet. In the vocabulary of grid libraries those are called offsets. Offsets allow you to 'skip' columns in your layout. Lets make some.

Step 05:  Create classes to make offsets. The easiest way to do this would be to take advantage of the fact that our pips will wrap if we exceed the space available on the row. Everytime we use 8 columns worth of space our elements will wrap. We only have to worry about the gap on the left of each object. To move something a little to the right...we can simply use the margin-left property. My 2 column offset for example looks like this:


.offset-2 {
  margin-left: 25%;
}

After creating the necessary offsets (you will need more than one), apply them to the correct pips. If you have done this correctly your card should look like the image below.

Step 06:  It's very close now. All we really need to do is get our row to take up the whole height of the card, and space everything accordingly. With flexbox this is easy. Add a couple of lines to your row class like the following:


.row {
    display: flex;
    flex-flow: row wrap;
    align-items: center;
    align-content: space-between;
    height: 100%;
}

Voila! We have a playing card! And a library we could use to make almost any card we need.

Next steps

We now have a library good for one specific purpose. It would be nice though to have a resource we could use in a wide variety of situations. Let's make some changes and take advantage of the magic of SCSS to make our grid more general purpose.

Step 07:  In your .scss file from the grid assignment create 2 mixins. One that will be used for columns and one for offsets. Make sure to take in a parameter for which column you need. In my case the mixins would look like this: 


@mixin col($span-num) {
    flex: 0 0 12.5% * $span-num;
}

@mixin offset($offset-num) {
    margin-left: 12.5% * $offset-num;
}

and to use them I would do this:


.col-1 {
    @include col(1);
}

This is a good start. Now if we need to adjust the css that makes our cells we can do it in one spot instead of 8. I think we can do more. As I look at the scss above I'm noticing that more than the contents of my column classes are the same...the classes themselves follow a pattern: col-[column span number]. This looks like an excellent place to take advantage of another feature of Sass: loops.

Loops in Sass work very similarly to other programming languages and in Sass you have either for loops, for-each loops, or while loops. For more information check out Jump Start Sass Ch5. In this case I think the for loop is the right choice.

Step 08:  Let's save ourselves some typing, and make our grid more flexible by re-factoring our SCSS to use a for loop to generate the column and offset classes.

Again I'll give an example of what this would look like using my code.


//make column classes 
@for $index from 1 to 9 {
    .col-#{$index} {
        @include col($index);
    }
}

//make offset classes
@for $index from 1 to 9 {
    .offset-#{$index} {
        @include offset($index);
    }
}

We start with the @for keyword, then we indicate the variable we want the loop to use while it is counting. Then we tell it where to start and stop counting. In this case we will start with 1 and end with 9. Why 9 you may be asking? The loop uses a less than (<) to check when it is done, and not a less than or equal to (<=) so it will stop before it does the 9th loop. If we left it at 8 we would be one short.

The next thing you may have noticed is the funny syntax around the class names: .col-#{$index}. You probably recognize our $index variable, but why is it wrapped in #{ }?

The short answer is because it won't work if we don't :) The better answer can be found here: All you ever need to know about SASS Interpolation.

With those for loops in place we can actually go back and remove all of the .col-1, .col-2, .offset-2, etc classes from our scss. The for loops will generate those for us. If you don't believe me go take a looks at your compiled .css!

At this point we have dramatically shortened our SCSS, and made it a bit more maintainable. We're still not finished however, it would be nice if we could use our grids for any number of columns instead of just one set. We have actually laid most of the groundwork to make this happen. Lets finish.

Step 09:  Add the appropriate variables to allow the grid to be used for any number of columns.

If we look back at the mixins we made earlier you will see a hard coded number: 12.5%. Lets make it a variable. 


$one-column: 12.5%;
   @mixin col($span-num) {
    flex: 0 0 $one-column * $span-num;
}

@mixin offset($offset-num) {
    margin-left: $one-column * $offset-num;
}

That is a good start, but we can do more. The 12.5% value in my mixins comes from my 8 columns (100% / 8). Lets modify our variable to reflect the formula used to determine it.


//variables
$num-columns : 8;
$one-column : 100% / $num_columns;

With that one change we are actually almost done. We can now indicate how many columns we want, and the appropriate size of one column will be calculated. the last thing to fix is the start and stop points for the loops. More variables!


//variables
$num-columns : 8;
$one-column : 100% / $num_columns;
$loop-start: 1;
$loop-end: $num-columns +1;

...and then the loops would need to change as well:


//make column classes 
@for $index from $loop-start to $loop-end {
    .col-#{$index} {
        @include col($index);
    }
}

//make offset classes
@for $index from $loop-start to $loop-end {
    .offset-#{$index} {
        @include offset($index);
    }
}

Step 10: And there we are. Our grid might not be complete enough to work in all situations, but we have made a lot of progress on making it more efficient and reusable.

There are two last steps we should do. First go back and do some cleanup on your code. Make sure that each mixin and variable is declared only once and make sure to remove any unnecessary scss. Then take the classes, loops, and mixins needed for your grid and create a Sass partial with them. That way it will be easy to incorporate into another project should the need arise.
