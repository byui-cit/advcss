Optimize your grid

Making a layout for a specific design can be challenging, after last week you have probably seen that making a library that will work in a wider variety of situations is even trickier. We made a grid that would work for one purpose...creating a playing card. This week I want to spend a few minutes to see if there are things we could do to make our code useful in a wider variety of situations.

Activity

This activity will have you continue working with the css grid library you created last week. For my examples I will be using the code that I came up with...it's ok if your code is different from mine. As the old saying goes...there is more than one way to skin a cat :)

Step 01:  Create a new pen in codepen.io. Make sure to change the css settings to use SCSS so you can take advantage of Sass. You should be able to use the same classes you created last week for the card backs to create the general shape of the card. Choose a card to start with (I recommend 5, 7, or 10) and add all the elements you will need.

Step 02:  Create the rest of the classes you will need to make your card look right. Then create a second different card to test your library. Hopefully you won't need to make any changes to your css to get it to work, but make adjustments as necessary. Once the first and second card are correct try as many others as you need to feel comfortable that your library will work for any card A-10. (Don't forget to try the Ace ;) )

Step 03:  Submit the URL to your codepen to Ilearn.

Grading

  • Grid library can be used to create any playing card A-10 by changing the html markup but without changing the css. 10pts
  • SCSS has been used where appropriate. (variables for colors or sizes would be a good start) 4pts
  • Grid is Flexbox based. 6pts
