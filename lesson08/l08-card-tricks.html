I was about 13 when I received a book all about card tricks and how to perform them. It was pretty exciting and I immediately dove in to start learning them. I made it through about 3 tricks before I realized card tricks could be really hard and took a lot of practice! Well unfortunately, I let that dampen my enthusiasm and I stopped practicing, that's why to this day I can only do about 2 tricks...but they are pretty cool :)

Most of life is like this actually. New and exciting until we realize that meaningful things usually take a lot of work! We are about at the point in the semester where the shine is wearing off. Hopefully we can keep our enthusiasm up and endure unto the end.

Activity

For this activity you will build your own card trick with CSS transitions and animations. You should pick 2 cards (A-10) and upon the hovering or clicking of one it will transform (magically) into the other.

Step 01:  Begin this activity by choosing 2 cards...one you will start with and one you will end with. Using your card grid system from last week build the starting card in html.

Step 02:  Using CSS transitions and animations (and maybe a little bit of transforms) magically convert the beginning card to the ending card in any way you would like.

Tips

  • Start by just getting the hang of making things transition or animate. Make things move around and appear/disappear.
  • If you need inspiration remember to check out Animatable
  • CSS Transforms can be really helpful for this...but that is next weeks topic! Because we haven't studied them yet I put together a few mixins to make it easy to use some of the more common transforms. 
    
//rotates an element $angle degrees  example of usage: @include rotate(180deg);
@mixin rotate($angle) {
    -webkit-transform: rotate($angle);
    transform: rotate($angle);
}
//grows or shrinks an element.  If you only pass in one arg it will scale x and y evenly, if you pass 2 it will scale accordingly.  Example: @include scale(1.5); (Would make it 1.5 times larger)
@mixin scale($x, $y:$x) {
    -webkit-transform: scale($x,$y);
    transform: scale($x, $y);
}
//Moves something horizontally by $x and vertically by $y. One arg will move it equally in both directions. Example @include translate(10px, 15px); (move 10px to the left and 15px down)
@mixin translate($x, $y: $x) {
    transform: translate($x, $y);
}
  • As you build out your css watch for places where SCSS can make things easier through variables, nesting, mixins, or maybe even loops.
  • If you want to make your trick trigger from a click you will need a bit of Javascript...the Chapter on transitions in CSS Master had a good example of this using JQuery or W3 Schools has an example of how you might do it without JQuery.
