CSS Code Review

We've been reading and talking a lot this week about css organization and architecture. We've only scratched the surface. This is an issue that every developer faces, and there are about as many opinions on the correct way to do it as there are developers...or projects...maybe even developers X projects. That's how many opinions there are ;)

You will eventually develop your own opinions as well. This activity will hopefully help you begin to do that by reviewing the css for a popular project...Foundation.

Activity

Based on the reading and discussions of this week about organization and architecture we will review the source code (css) for the Foundation framework and discuss what they have done to organize their code.

Step 01:  There are 2 ways you can look at the code for the Foundation framework (you should look at the SCSS version and not the compiled CSS version). You look at the SCSS version on github, or if you are feeling adventurous you can follow the instructions for Foundation Installation to pull a copy down to your computer. If you choose the install I would follow the manual:basic setup instructions. Choose one and look at the code.

Step 02:  Review Foundation and write-up your thoughts considering the following as you review:

  • Is the styling is de-coupled from the html structure? Try using a few components in a page like button, topbar, or progress bar. (If you visit Css settings in codepen you can quickly add Foundation as an External CSS resourceâ€¦see the Quick-add box) Can you use them by just using classes or do they require very specific markup to work?
  • Which if any of the philosophies we discussed this week they might be using (BEM, Atomic, OOCSS, etc)?
  • Are the folders they use well named?
  • Check out the settings partial. Imagine trying to customize Foundation without that settings file! Consider the advantages Sass gives to both the organization and customization of Foundation.
  • If you were a developer working on this code and you were asked to work on on something specific...like headline styles...how hard would it be to locate the code and modify it? Would you worry about your changes having unintended consequences after reviewing the class anmes and architecture of that portion of css?
  • Do you like how it is organized? Would you change anything?

Step 03:  Submit your notes to Ilearn.

Grading

  • Thoughtful review of the code given. 5pts
  • At least 250 words. 5pts
