Bootstrap

Bootstrap advertises itself as the "world's most popular front-end component library". I can't speak to the veracity of that statement, but I do know it is pretty cool.

A framework is really just a collection of code (in this case html, css, and javascript) and design patterns that solve common design problems. Bootstrap is no exception. It started out as a styleguide for Twitter.

Most projects have the same core set of issues that have to be addressed. One huge advantage of css frameworks (or any framework for that matter) is that they have solutions pre-built and more importantly very well tested to those core problems. Thus they can save developers a lot of time.

Activity

Bootstrap gives several options for use. The simplest would be to simply download the full, compiled CSS file, and link it into your html normally. This is a great option to get up and going quickly, but is not really great for production. Most sites will not use all of what Bootstrap offers and so downloading the full CSS is overkill.

The next option would be to take advantage a CDN (content delivery network). This is a server optimized for speed that has one job...serve up libraries. Bootstrap's CDN can be included by adding the following lines to your html 


<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.1.3/dist/css/bootstrap.min.css" rel="stylesheet" integrity="sha384-1BmE4kWBq78iYhFldvKuhfTAU6auU8tT94WrHftjDbrCEXSU1oBoqyl2QvZ6jIW3" crossorigin="anonymous">
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.1.3/dist/js/bootstrap.bundle.min.js" integrity="sha384-ka7Sk0Gln4gmtz2MlQnikT1wXgYsOg+OMhuP+IlRH9sENBO0LRn5q+8nbTov4+1p" crossorigin="anonymous"></script>

Bootstrap can also be quickly integrated into a codepen. If you click on the CSS setting (gear icon next to CSS) then scroll down to the "Add External CSS section" you will see a drop down labeled "Quick-add". Bootstrap is one of the options there. This can let you quickly try something out.

The best option however is to take advantage of Bootstrap's Sass version. The rest of this activity will have us install Bootstrap into a project with a professional style workflow. We will be roughly following the instructions in the Bootstrap documentation

Step 1

Create a folder on your computer called parcel. Open that folder in your editor (this activity will be easier if you are using VS Code as your editor). If you are not using VS Code I would recommend creating this folder on your Desktop so it will be easier to find.

If you are using VS Code as your editor you can open the terminal in your open project (ctrl + ~) and it will automatically set the working directory to the open project folder. If you are using some other editor, Open your command line prompt on your computer and navigate to the directory containing the downloaded code. Mac and Windows should start you in your user directory when you open a new command prompt. Or you can type:

  • Mac: 
    cd ~
    (tilde) or
  • Windows: 
    cd %userprofile%

from anywhere and it will take you back to your user directory. From the User directory you should just need to type 

cd Desktop
then 
cd parcel

to get to the right place. Then type

npm init -y

This will create a file called package.json. This file will contain the details about our project.

Modern web projects rely on a few types of tools to make them easier to manage.

Package Managers
This tool controls all of the dependencies for the project...it lets us easliy include tools written by others to both help us in development, and to include additional functionality in our finished product. It really helps when it's time to update to a newer version of one of these tools.
Asset Bundlers
These gather all of the assets that our project needs and consolidates and processes them. A web app for example might have dozens of javascript files containing it's code. The bundler will combine them all together and minimize them...and then move the final file into a folder ready for distribution.
Task Manager
There are many processes that need to be run each time a change is made to the project. Sass needs to compiled and possibly autoprefixed and minimized, javascript combined and maybe transpiled, etc. A task manager can be configured to allow all of these tasks to be automated.

For this project NPM is our package manager. In fact when you ran npm init we did the first step in making this happen.

We are using a tool called Parcel for bundling. Other popular bundlers would be webpack or browserify

Finally since our project is fairly simple we are actually using NPM as our task manager as well. If you look again in the package.json file you will see a section called scripts. We will place commands there to do different things for our project. Other popular task managers are Grunt and Gulp

Step 2

Install Parcel by typing npm install --save-dev parcel (make sure that you are in the same directory that the package.json file is in) in the terminal. This will add Parcel as a development dependancy to our project...a tool that we will use as developers to do our work. You should notice that a couple of things changed: first a new folder called node_modules was created, second a new section called devDependencies was added to our package.json.

Next we need to add some scripts to allow us to use Parcel. Add the following code to the scripts section of your package.json 


"scripts": {
    "start": "parcel ./src/index.html",
    "prebuild": "npx rimraf build",
    "build": "parcel build --public-url ./ ./src/index.html  --out-dir build"
},

Step 3

The Bootstrap instructions recommend the following file structure: 

project-name/
├── build/
├── node_modules/
│   └── bootstrap/
│   └── popper.js/
├── scss/
│   └── custom.scss
├── src/
│   └── index.html
│   └── index.js
└── package.json

The build directory will get created automatically for us later, but we should create the other folders and files now.

  1. Create a scss directory in the root of the project folder, then add a custom.scss file inside of that directory.
  2. Create a src directory in the root of the project folder and then create files called index.html and index.js in that folder.
  3. Open up the index.html file and add the following code: 
    
<!doctype html>
<html lang="en">
    <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <link rel="stylesheet" href="../scss/custom.scss" >
    </head>
    <body>
    <h1>Hello Bootstrap</h1>
    <p class="alert alert-primary">Demo site illustrating using a build tool to install Bootstrap</p>
    <script src="./index.js" type="module"></script>
    </body>
</html>

Notice the file we linked to for our styles. We linked directly to the .scss file. That should not work! Browsers do not understand .scss! Our bundler Parcel needs the path to the SCSS so it can find it, and it will automatically convert the SCSS to CSS and update the link in the index.html for us when it is done.

Test our installation by running the following

npm run start

in the command line of your computer. Make sure you are still in the project folder. (the same folder that contains the package.json file) This will start up a simple web server pointed at our project. It will also watch for changes to either the .SCSS files or the .JS files and it will recompile them when needed and refresh the page in the browser. It is not watching the .HTML file however. So if you make changes to that you will need to refresh the browser manually.

To see your site notice that Parcel output Server running at http://localhost:1234 when it started up. Your port number (ie :1234) will probably be different. Enter that url in your browser to see it.

When you are ready to stop the web server type ctrl+c in the command line.

Step 4

Next let's install Bootstrap using our package manager. Go back to your command prompt and quit the webserver if it is running (ctrl+c). Then type 

npm install bootstrap

The add command retrieved the packages and placed them into the node_modules directory, but our project still doesn't know about them. Lets do that next.

Step 5

Finish adding Bootstrap into your project by opening the src/index.js file in an editor and adding the following line at the very top: 


// Import all plugins
import * as bootstrap from 'bootstrap';

Those lines will make it possible for our project to use the libraries. Notice that we didn't give much information about where the actual files were located...our bundler is smart enough to find them as long as they are mentioned in package.json and installed in node_modules.

Then open up the /scss/custom.scss file and add the following line just under the @import for the variables partial: 


@import "../node_modules/bootstrap/scss/bootstrap";

We imported ALL of Bootstrap with that line. That is not always the best thing to do, we should really only import the parts of Bootstrap we are using. Check out the Bootstrap docs on importing to see how you can be more selective about what you include.

Ok, lets try firing up our development server again. Type npm run start. Your page should have picked up a bit more styling...different font, and the alert should be styled. If you see that then it's working! If not then review the directions above, or ask for help.

Take a minute to notice that a new folder was created by Parcel called dist and check out the files that were generated: dist/index.?????.css and dist/index.??????.js. Notice that they are each over 200KB now! Over 400KB of stuff just to make Bootstrap work...that is a lot. The FamilySearch homepage has a total budget of ~60KB for all of it's html, css, and Javascript for example!

We did include all of Bootstrap however. A better option would be to only include the parts we intend to use.
