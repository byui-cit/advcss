Bootstrap

Bootstrap advertises itself as the "world's most popular front-end component library". I can't speak to the veracity of that statement, but I do know it is pretty cool.

A framework is really just a collection of code (in this case html, css, and javascript) and design patterns that solve common design problems. Bootstrap is no exception. It started out as a styleguide for Twitter.

Most projects have the same core set of issues that have to be addressed. One huge advantage of css frameworks (or any framework for that matter) is that they have solutions pre-built and more importantly very well tested to those core problems. Thus they can save developers a lot of time.

Pre-requisites

There are several utilities you will need to make sure are working on your computer for the install to go smoothly. If you followed the software setup instructions you should be most of the way there. Here is a list of things to verify. Open a command line window and try the following:

  • Make sure that NPM (Node Package Manager) is working by entering: 
    npm -v
    You should see a version number. If you get any errors or "command not found" try running through the software setup instructions again.
  • Install Yarn (a javascript package manager) by entering 
    npm install -g yarn
    .

If you run into any issues get some help from a fellow student or the instructor.

Activity

Bootstrap gives several options for use. The simplest would be to simply download the full, compiled CSS file, and link it into your html normally. This is a great option to get up and going quickly, but is not really great for production. Most sites will not use all of what Bootstrap offers and so downloading the full CSS is overkill.

The next option would be to take advantage a CDN (content delivery network). This is a server optimized for speed that has one job...serve up libraries. Bootstrap's CDN can be included by adding the following lines to your html 


<link rel="stylesheet" href="https://maxcdn.bootstrapcdn.com/bootstrap/4.0.0/css/bootstrap.min.css" integrity="sha384-Gn5384xqQ1aoWXA+058RXPxPg6fy4IWvTNh0E263XmFcJlSAwiGgFAW/dAiS6JXm" crossorigin="anonymous">
<script src="https://maxcdn.bootstrapcdn.com/bootstrap/4.0.0/js/bootstrap.min.js" integrity="sha384-JZR6Spejh4U02d8jOt6vLEHfe/JQGiRRSQQxSfFWpi1MquVdAyjUar5+76PVCmYl" crossorigin="anonymous"></script>

You would also need to include JQuery for this to work.

Bootstrap can also be quickly integrated into a codepen. If you click on the CSS setting (gear icon next to CSS) then scroll down to the "Add External CSS section" you will see a drop down labeled "Quick-add". Bootstrap is one of the options there. This can let you quickly try something out.

The best option however is to take advantage of Bootstrap's Sass version. The rest of this activity will have us install Bootstrap into a project with a professional style workflow.

Step 1

Download the Boilerplate code and extract it to your Desktop on your hard drive

Open your command line prompt on your computer and navigate to the directory containing the downloaded code. Mac and Windows should start you in your user directory when you open a new command prompt. Or you can type:

  • Mac: 
    cd ~
    (tilde) or
  • Windows: 
    cd %userprofile%

from anywhere and it will take you back to your user directory. From the User directory you should just need to type 

cd Desktop
then 
cd webpack

to get to the right place. Then type

yarn install

It might take a few minutes

Attention!

If you get an error complaining about your Node being incompatible with a package then you will need to use this alternate syntax: 

yarn install --ignore-engines

Once you have it installed go take a look at the file structure. The boilerplate that you received is setup similar to what you might find with any small to medium scale modern web project. Modern development projects end up with a lot of moving parts. There are dependencies...other utilities or libraries that are being used to speed up things. Some of those have special requirements as well. Sass would be one example...it needs to be compiled before it can be used.

Modern web projects rely on a few types of tools to make them easier to manage.

Package Managers
This tool controls all of the dependencies for the project...it lets us easliy include tools written by others to both help us in development, and to include additional functionality in our finished product. It really helps when it's time to update to a newer version of one of these tools.
Asset Bundlers
These gather all of the assets that our project needs and consolidates and processes them. A web app for example might have dozens of javascript files containing it's code. The bundler will combine them all together and minimize them...and then move the final file into a folder ready for distribution.
Task Manager
There are many processes that need to be run each time a change is made to the project. Sass needs to compiled and possibly autoprefixed and minimized, javascript combined and maybe transpiled, etc. A task manager can be configured to allow all of these tasks to be automated.

For this project Yarn is our package manager. In fact when you ran yarn install it went and looked at the package.json file to see what dependencies the project has, then retrieved them and put them into a directory called node_modules. Another popular package manager is npm

We are using a tool called Webpack for bundling. There is a configuration file for it as well: webpack.config.js. Another popular bundler would be browserify

Finally since our project is fairly simple we are actually using Yarn as our task manager as well. If you look again in the package.json file you will see a section called scripts. these are commands we can run to do different things for our project. Other popular task managers are Grunt and Gulp

Step 2

To test your installation type the following:

yarn start

in the command line of your computer. Make sure you are still in the project folder. This will start up a simple web server pointed at our project. It will also watch for changes to either the .SCSS files or the .JS files and it will recompile them when needed and refresh the page in the browser. It is not watching the .HTML file however. So if you make changes to that you will need to refresh the browser manually.

When you are ready to stop the web server type ctrl+c in the command line.

You should see a simple unstyled button and a message: Setup Appears to be working ðŸ˜‰

Step 3

Next let's install Bootstrap using our package manager. Go back to your command prompt and quit the webserver if it is running (ctrl+c). Then type 

yarn add bootstrap jquery popper.js

Attention!

If you got the Node error earlier you will need to run the following command instead: 

yarn add --ignore-engines bootstrap jquery popper.js

You may be asking yourselves: "I thought we just needed Bootstrap? Why Jquery and popper.js?" In order to use many of the Bootstrap components Bootstrap requires those two javascript libraries in order to function. Luckily they are easy to add with a package manager.

The add command retrieved the packages and placed them into the node_modules directory, but our project still doesn't know about them. Lets do that next.

Step 4

Finish adding Bootstrap into your project by opening the assets/js/index.js file in an editor and adding the following lines at the very top: 


import $ from "jquery";
import popper from 'popper.js';
import bootstrap from 'bootstrap';

Those lines will make it possible for our project to use those libraries. Notice that we didn't give much information about where the actual files were located...our bundler is smart enough to find them as long as they are mentioned in package.json and installed in node_modules.

Then open up the assets/scss/styles.scss file and add the following line just under the @import for the variables partial: 


@import "node_modules/bootstrap/scss/bootstrap";

Ok, lets try firing up our development server again. Type yarn start. Your page should have picked up a bit more styling...different font, button styled, and you should see one more message indicating that Jquery is working. If you see that then it's working! If not then review the directions above, or ask for help.

Take a minute and check out the files that were generated: assets/css/styles.css and assets/js/bundle.js. Notice that they are each around 150KB now! Over 300KB of stuff just to make Bootstrap work...that is a lot. The FamilySearch homepage has a total budget of ~60KB for all of it's html, css, and Javascript for example!

We did include all of Bootstrap however. A better option would be to only include the parts we intend to use.

If you look inside of the node_modules/bootstrap/scss directory you will see a long list of partials. These could be included individually instead of all at once if we wanted.
