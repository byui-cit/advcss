CSS Module: SVG Card backs

Playing cards have been around in one form or another for a long time. They traditionally have very intricate patterns on the backs some of which are truly works of art. If you haven't looked at the back of a deck of cards recently take a minute and do a quick search.

Activity

This activity will have you create 2 custom playing card backs. Both should include multiple shapes and should be inline SVG (right in the html). One of the cards should be entirely your own work. For the other you are welcome to incorporate SVG shapes you have found on the web.

Step 01:  Create a new pen in codepen.io. Make sure to change the css settings to use SCSS so you can take advantage of the things we learned last week. I would recommend starting with a simple rectangle with slightly rounded corners to define your card. Hint: the height of a card is generally 1.4X the width.

Step 02:  Create your cards, when completed submit the url to your pen in ILearn.

If you have issues with scaling your svg objects take another look at the article: Understanding SVG Coordinate Systems along with the playground that goes along with the article.

You may run into some strange scaling issues if you use symbols with patterns. When you create a viewBox you essentially redefine the coordinate system the svg uses. If the viewBox is smaller than the viewPort (the size of the SVG element) then scaling will occur. Here is a codepen with some examples of some of the behaviors you might see because of this. Make sure to read the comments.

This assignment will also ask you to group some of your svg objects. There are some examples of this in the SVG Shapes activity you should have completed, but here is another article that goes into more depth to review: Structuring, Grouping, and Referencing in SVG

Completion

  • Two card backs with interesting patterns created.
  • SCSS has been used where appropriate. (variables for colors or sizes would be a good start)
  • Both backs use at least 3-4 shapes.
  • Some sort of SVG grouping has been used (g, pattern, symbol, etc).
  • CSS has been used to style some portion of the SVG.
