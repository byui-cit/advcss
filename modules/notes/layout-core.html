Layout Core teaching notes

CSS Layout

I would begin by reviewing the Smashing magazine Layout article. Make sure the students understand Normal flow. Then talk about position. This layout technique is often highly discouraged...mainly because beginning students use it wrong! Make sure they understand how the coordinate systems work...ie position absolute coordinates will be top/left corner of first non-static ancestor.

This opens us up to understand how to appropriately use position:absolute. It should almost always be used inside of a position: relative container.

Grids

Move on to grids...talk about how there is some confusion because 'grid' is overused as a word in CSS. It could be used to refer to 'designing to a grid', a 'grid library', or 'CSS grid'. Three related but different things. Review this grid theory article.

Which one should I use!?

It depends :) But most of the time now your layouts should be a combination of CSS Grid and Flexbox. If you run into a layout problem where they don't work, then look at the others...

Intrinsic designs

Finally spend a bit of time discussing the contents of the Jen Simmons presentation: Designing Intrinsic Layouts

In earlier courses students are taught that fixed units are bad. They should not use px to determine spacing for example because it's not responsive. Sometimes however this causes problems, columns that are too large or too small...or both! Sometimes it's OK when we know how big something should be to just make it that big. Then you handle responsiveness through the rest of the page. Jen simmons calls this mix of fixed and responsive elements in a layout Intrisic Design.
