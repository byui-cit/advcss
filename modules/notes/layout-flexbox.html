CSS Flexbox fundamentals

Flexbox is all about one dimensional flows. We have great control over one direction (or axis) but not so much on the other. If we need exact placement along both directions...then Flexbox is not what we need. A large number of websites fit within this however. You could reference the Jen Simmons notes below for examples.

I always start with boxes: Boxes codepen. It's helpful to pull up the CSS Tricks: Guide to Flexbox for reference. I work down the Parent side first, then down the child. Understanding primary/secondary Axis, and alignment tends to be the trickiest for students to understand.

Justify always affects the primary axis, Align always affects the secondary axis. MDN: Box Alignment in Flexbox

With Children understanding the flex property tends to be the hardest. I often intentionally stop using the abbreviated flex, and instead break it down into flex-grow, flex-shrink, and flex-basis until students understand what each part does.
