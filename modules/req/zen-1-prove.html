CSS Zen Garden Style Guide

By now you should have formed groups and picked a theme that you will use for your version of the CSS Zen Garden. You should have reviewed the content and prepared a couple of wireframes as well, making sure to include everything. This activity will have you continue your work on this project by continuing your planning for your version of the site.

The chapter from Atomic Design that you were asked to read presented the idea that there is a lot of merit in approaching web pages or sites as collections of components...modular design. It then goes further and proposes applying structure and documentation through the creation of a detailed vocabulary and of a style guide or pattern library as part of the development process.

You should approach your design of your Zen Garden site from this direction.

Activity

  1. Read Atomic Design Ch2

    Read further in the Atomic Design book. Read enough of Ch2 that you feel like you have a good feel for the concepts.

  2. Identify parts

    Go back and review a few different versions of the Zen Garden. Identify and make a list of the different atoms, molecules, and organisms that you see. As you switch versions watch for patterns and use them to help you identify the parts.

  3. Apply your theme

    With the list from the previous step, review your chosen theme and start thinking about how you will adapt your it to style the atoms, molecules, and organisms identified in the previous step.

  4. Begin a Styleguide/pattern library

    You should complete the Preprocessors Module before continuing.

    Begin your styleguide by creating a few scss files. Let's create one for atoms, one for molecules, and one for organisms to start. Then create a file called style.scss. This will be your main scss file for the project. We need to bring all of these files together so they can be seen by our page.

    SCSS modules will work great for this. It would look something like this: 

    // styles.scss
@use 'atoms';  // assumes file is named _atoms.scss
@use 'molecules';
@use 'organisms' as o;  // can shorten the name if we like :)

  5. Choose colors and fonts

    Give some thought to your color scheme and fonts. Consider your theme as you do this.

    Create another SCSS module called _variables.scss. Add variables to that file with your chosen colors and fonts.

  6. Create atoms

    Create at least 5 atoms for your project. These should be for base styling for your version of the Zen Garden. For example take a look at the Screen Filler version of the Zen Garden. The H1 has a definate style. That would become one of your atoms.

    Also in the header we have a couple of paragraphs that are all uppercase. That set of styles is used at least three times in the design. I would add a mixin to create that look. Look for a similar opportunity to use mixins in your plans.

  7. Review this example

    Using the Zen version mentioned above here is an example of how your style guide/pattern library could come together using SCSS. Review it and make adjustments to yours as necessary. Pattern library example. Here is a link to the source SCSS files for the example.

    Note that this is just one way to structure this. In a more complete Atomic implementation you might see many atom files. Sometimes as many as one file per atom. Feel free to experiment with this. As you are working with a group to complete this assignment you might find that more SCSS modules will make your job easier than just one or two.

Grading

  • At least three SCSS modules created.
  • SCSS compiles without errors.
  • Principles of Atomic design have been applied.
  • A minimum of the following sections completed: theme, colors, typography, and at least 5 atoms (elements) defined including at least one mixin.
