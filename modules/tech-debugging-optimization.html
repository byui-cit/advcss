Tech Module: Debugging and Optimization

Hours: 6

Prepare

Review the following resources on debugging and optimizing CSS.

  1. Make sure that you have completed the software setup activity from Week 1.
  2. Read CSS Master (3rd Edition) Ch 3 (Sitepoint).
  3. Study Jump Start Web Performance Chapters 1-2 (Sitepoint).
  4. Watch Selector Efficiency and Simplification from the Architecture course on Sitepoint (14 min).
  5. Review Google Chrome DevTools Documentation You are not expected to read through all of this. I would recommend the CSS and Accessibility sections as a starting point. (00:11 mins, Chrome DevTools: Changing A Page's Background Color Transcript)
  6. Review the resources at MDN: Web Performance pay special attention to the sections on Perceived Performance and CSS Performance.

Ponder

Complete the following activities:

  1. Complete CSS Troubleshooting in Six Easy Steps SitePoint course (54min). Some of this course will be review, but make sure to check out lessons 1, 4, and 5 at a minimum.
  2. CSS Utilities.

    Make sure that you are familiar with how to use the command prompt for your operating system. Here are some quickstarts for the command line for Mac and the command line for Windows if you need a review.

Prove

Complete the following activities:

  1. Choose one of the testing tools from the Jump Start Web Performance book. (The Lighthouse tool in Chrome is actually a great place to start if you are unfamiliar with it.) Run the tool against a website of your choice or the BYU-Idaho website and review the results. Make sure you understand each of the issues it brings up and do enough research to know how you might go about correcting the issue. Write up your recommendations for improving the performance of the site.
