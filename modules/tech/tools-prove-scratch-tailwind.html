Front end Development Workflow with Tailwind.

TailwindCss advertises itself as "A utility-first CSS framework...". It has rapidly grown in popularity among web developers and is worth taking a look at.

A framework is really just a collection of code (in this case html, css, and javascript) and design patterns that solve common design problems. Utility classes have always existed in CSS. they are simpole classes that do one thing and one thing only. A team will create them on a project by project basis as needed. Tailwind takes utilities to the extreme. Essentially everything is a small focused utility class in TailwindCss.

Tailwind has been a bit polarizing. It does definately bloat up your HTML with lots and lots of classes. It's proponents however say that it is worth it for the speed at which they can style pages and components once they learn it.

Activity

Tailwind is actually built or compiled similar to SCSS. Because of this you need to install the tailwind application in order to use it. This activity will show one way of setting up a new project that uses Tailwind.

Step 1

Create a folder on your computer called tailwind-demo. Open that folder in your editor (this activity will be easier if you are using VS Code as your editor). If you are not using VS Code I would recommend creating this folder on your Desktop so it will be easier to find.

If you are using VS Code as your editor you can open the terminal in your open project (ctrl + ~) and it will automatically set the working directory to the open project folder.

If you are using some other editor, Open your command line prompt on your computer and navigate to the directory containing your created directory.

Mac and Windows should start you in your user directory when you open a new command prompt. Or you can type:

  • Mac: 
    cd ~
    (tilde) or
  • Windows: 
    cd %userprofile%

from anywhere and it will take you back to your user directory. From the User directory you should just need to type 

cd Desktop
then 
cd tailwind-demo

to get to the right place.

Once you are in the right directory type:

npm init -y

This will create a file called package.json. This file will contain the details about our project.

Modern web projects rely on a few types of tools to make them easier to manage.

Package Managers
This tool controls all of the dependencies for the project...it lets us easliy include tools written by others to both help us in development, and to include additional functionality in our finished product. It really helps when it's time to update to a newer version of one of these tools.
Asset Bundlers
These gather all of the assets that our project needs and consolidates and processes them. A web app for example might have dozens of javascript files containing it's code. The bundler will combine them all together and minimize them...and then move the final file into a folder ready for distribution.
Task Manager
There are many processes that need to be run each time a change is made to the project. Sass needs to compiled and possibly autoprefixed and minimized, javascript combined and maybe transpiled, etc. A task manager can be configured to allow all of these tasks to be automated.

For this project NPM is our package manager. In fact when you ran npm init we did the first step in making this happen. Another popular package manage is PNPM

We are using a tool called Vite for bundling. Other popular bundlers would be webpack or Parcel

Finally since our project is fairly simple we are actually using NPM as our task manager as well. If you look again in the package.json file you will see a section called scripts. We will place commands there to do different things for our project. Other popular task managers are Grunt and Gulp

Step 2

Install Vite by typing npm install -D vite (make sure that you are in the same directory that the package.json file is in) in the command line. This will add Vite as a development dependancy to our project...a tool that we will use as developers to do our work. You should notice that a couple of things changed: first a new folder called node_modules was created, second a new section called devDependencies was added to our package.json.

Next we need to add some scripts to allow us to use Vite. Add the following code to the scripts section of your package.json 


"scripts": {
    "start": "vite",
    "build": "vite build",
    "preview": "vite preview"
},

Step 3

https://tailwindcss.com/docs/installation/using-postcss Vite is a very flexible tool that will accomodate a wide range of file structure in our projects. We will keep ours simple however. Once we have everything setup with Vite and Tailwind our file structure should look something like below. 

project-name/
├── dist/
├── node_modules/
├── css/
│   └── style.css
├── public
├── index.html
├── postcss.config.js
├── tailwind.config.js  
└── package.json

The dist directory will get created automatically for us later, as well as the postcss.config.js and tailwind.config.css, but we should create the other folders and files now.

  1. Create a css directory in the root of the project folder, then add a style.css file inside of that directory.
  2. Create a public directory in the root of the project folder that will contain images and other static assets
  3. Create an index.html file and add the following code: 
    
<!doctype html>
<html lang="en">
    <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <link rel="stylesheet" href="css/style.css" >
    </head>
    <body>
    <h1 class="text-3xl text-red-500 underline">Hello world!</h1>
  
    </body>
</html>

Test our installation by running the following

npm run start

in the command line of your computer. Make sure you are still in the project folder. (the same folder that contains the package.json file) This will start up a simple web server pointed at our project. It will also watch for changes to either the .CSS files or the .JS files and it will recompile them when needed and refresh the page in the browser.

To see your site notice that Vite output ➜ Local: http://localhost:5175/ when it started up. Your port number (ie :5175) will probably be different. Enter that url in your browser to see it.

When you are ready to stop the web server type ctrl+c in the command line.

Step 4

Next let's install Tailwind using our package manager. Go back to your command prompt and quit the webserver if it is running (ctrl+c). Then type 

npm install -D tailwindcss postcss autoprefixer

After that has finished run one more command:

npx tailwindcss init -p

This will create the two missing config files for us.

You may be asking yourselves: "I thought we just needed Tailwind? Why postcss and autoprefixer?" Postcss is a very handy plugin manager that will allow us to do all sorts of things to our CSS. We will use it to compile our Tailwind code to CSS and then to automatically apply any prefixes that we might need as well.

We could also apply plugins that would allow us to use nesting without SCSS, or use SCSS style import statements.

The add command retrieved the packages and placed them into the node_modules directory, but our project still doesn't know about them. Lets do that next.

Step 5

Finish installing Tailwind into your project by opening the tailwind.config.js file in an editor and changing the line with the content 


            content: ["./*.{html,js}"],

Those lines will make it possible for Tailwind to know which files to look at when it is building the final CSS file. We basically tell it to "start in the current directory (./), then look for any files (*) that have a .js or .html extention (.{html,js})".

One last step. Open up the style.css file and add the following lines: 


@tailwind base;
@tailwind components;
@tailwind utilities;

Ok, lets try firing up our development server again. Type npm run start. Your page should have picked up a bit more styling...red, a different font. If you see that then it's working! If not then review the directions above, or ask for help.

You may have noticed that we are still missing one folder that was mentioned above: dist. To get that folder we can run another command. First focus the terminal window (click in it), then press ctrl-c to make sure that Vite is not still running. Then type npm run build. That command tells Vite to build a production ready version of our code and it will place it in a directory called dist. If we were going to make this site live on a server that is the code we would move.

Open that directory and check out the files that were generated: dist/assets/index.?????.css. Why does it have a bunch of random characters in the name? Well good webservers in order to be efficient will cache things like CSS and Javascript files. So sometimes when you make updates you will see the old version of the file for a bit until the cache finally updates.

Adding the random characters on the build file is sometimes called cache busting. It is a new file and so will show up the first time you request it!
